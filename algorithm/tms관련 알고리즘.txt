🔧 1. 전통적인 최적화 알고리즘
    Dijkstra 알고리즘
        최단 거리(시간 또는 거리 기준)를 찾는 데 가장 많이 사용됨.
        정해진 맵에서 가장 빠른 경로를 찾을 때 적합.
    A* (A-Star) 알고리즘
        Dijkstra보다 효율적으로 동작.(다익스트라 확장)
        휴리스틱 함수(h(n))를 통해 목적지까지의 예측 비용을 고려.
    벨만-포드(Bellman-Ford) 알고리즘
        음의 가중치가 있는 그래프에서도 사용 가능.	
        
📦 2. VRP 계열 알고리즘 (Vehicle Routing Problem)
    TMS에서 실제로는 단순한 최단 경로보다, 복수의 차량이 여러 지점을 최적으로 방문해야 하는 문제에 가깝습니다. 이를 해결하기 위해 다양한 VRP 변형 알고리즘이 쓰입니다:
        CVRP (Capacitated VRP): 적재량을 고려한 차량 경로 최적화
        VRPTW (VRP with Time Windows): 각 고객이 받을 수 있는 시간대를 고려
        MDVRP (Multi Depot VRP): 여러 출발지(물류센터) 고려
        Fuel-Constrained VRP (FCVRP): 연료나 전력 제약 조건 반영
        
        이를 해결하기 위한 알고리즘은 다음과 같습니다:
        
    정확한(Exact) 방법
        Mixed Integer Programming (MIP)
        Branch and Bound
        Dynamic Programming
    휴리스틱 / 메타휴리스틱 방법
        Savings Algorithm (Clarke & Wright)
        Tabu Search
        Genetic Algorithm (GA)
        Simulated Annealing
        Ant Colony Optimization
        Particle Swarm Optimization
    
    대부분의 상용 TMS는 메타휴리스틱 기반 VRP 솔버 + 실시간 경로 재계산 엔진을 결합합니다.
